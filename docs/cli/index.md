## 基础概念

目前共有四个主命令，分别是：

  - `arcadia` 主要命令，用于导入配置、管理服务等
  - `task` 处理任务命令，用于执行脚本、管理脚本等
  - `update` 更新命令，用于更新项目、更新环境等
  - `envm` 变量管理命令，用于管理用户定义的环境变量

在接下来的文档内容和命令帮助菜单中，你会看到很多由尖括号括起来的内容，这些内容都是需要你自行输入的，比如：

  - `<xxx>` 表示需要你自行输入与命令相关的内容，会有提示

  - `<xxx/xxx>` 表示可选的自定义输入内容类型，你输入内容的类型可以是其中任意一个

  - `<args>` 表示固定可选的子命令，分别表示对应命令的不同用法

  - `[xxx]` 表示可选的命令选项即 `--options`，分别对应不同功能

需要注意的是尖括号是作为指导用户输入的提示，而不是实际的命令内容，所以你在输入命令时不需要输入尖括号

## 执行命令的位置

:::info 你需要知道的概念
执行位置区分容器内、容器外这一基础概念性问题，容器外即表示宿主机，由于项目运行在容器内，所以不能直接在宿主机执行相关命令，可通过如下两种方式实现
:::

### 进入容器内

```bash
docker exec -it <容器名称/容器ID> bash
```
后台管理面板中的命令行界面就是在容器内操作的

### 在宿主机调用

```bash
docker exec -it <容器名称/容器ID> bash <命令>
```

执行复杂命令建议使用 `docker exec -it <容器名称/容器ID> bash -c "<命令>"`，多个命令需用 `;` 或 `&&` 连接即 `"<命令> && <命令>"`  
后者 `&&` 表示前一条命令执行成功时再执行后一条命令，否则遇到报错会中断，相比前者用法较为特殊
