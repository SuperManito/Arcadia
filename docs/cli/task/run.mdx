## 执行方式

<Tabs>
<TabItem value="normal" label="普通" default>

```bash
task run <name/path/url>
```

</TabItem>
<TabItem value="concurrency" label="并发">

```bash
task conc <name/path/url>
```

并发是为每个账号作为单独的进程同时在后台运行

:::caution
并发执行非常消耗资源不要盲目使用，否则机器资源占满导致连不上终端，若想恢复也许只能操作强制关机重启
:::

</TabItem>

:::note 关于脚本运行日志
脚本运行的日志存放在 **log** 目录下的文件夹内，文件夹名为`脚本名去后缀`，日志文件名为日期，会以目录的形式进行分类  
目录名中的主要组成部分是脚本名称，非本地个人脚本的目录名称为`<仓库名/raw>_脚本名去后缀`，中间用下划线分割
:::

:::note 链接地址自动纠正是什么功能？
当拉取位于远程托管仓库的脚本时，无需刻意填写脚本文件的 raw 原始文件链接地址，可自动转换为原始文件链接以便于使用，此功能已应用到整个项目
:::

### 注意事项

1. 目前支持运行 `js` `py` `ts` `sh` 共四种类型的脚本，已默认预装了 Python 和 TypeScript 的运行环境，后续会适配更多类型的脚本
2. 当执行本地脚本文件时，脚本名的后缀格式（脚本类型）可以省略，届时将启用模糊查找，优先级为 `JavaScript` > `Python` > `TypeScript` > `Shell`，当存在同名脚本时仍适用此规则  
3. 如果执行本地的个人脚本则可以省略路径，如果执行的是已配置的脚本仓库或远程脚本中的脚本那么可以省略绝对路径而直接使用相对路径，例如 `<仓库目录名称>/template.js` `raw/template.js`

| :-------------------: | - | - | ----- |
| `--loop` / `-l`       | ✔ |   | 循环运行，连续多次运行脚本，参数后需跟`循环次数(正整数)`作为参数值，该参数与**等待执行**和**延迟执行**参数同时使用时仍然有效互不干涉 |
| `--mute` / `-m`       | ✔ | ✔ | 静默运行，静默运行任务不推送任何通知消息 |
| `--wait` / `-w`       | ✔ | ✔ | 等待执行，等待指定时间后再执行脚本，参数后需跟`等待时间单位`作为参数值，具体参照 [sleep](https://www.runoob.com/linux/linux-comm-sleep.html) 命令的用法 |
| `--hang` / `-h`       | ✔ |   | 后台挂起，将脚本设置为守护进程保持在后台运行，期间中断或结束会自动重新运行 |
| `--agent` / `-a`      | ✔ | ✔ | 网络代理，使脚本通过 HTTP/HTTPS 全局代理进行网络请求，目前仅支持 JavaScript 脚本，使用该功能需要自行在配置文件对应处定义代理地址变量 |
| `--delay` / `-d`      | ✔ | ✔ | 延迟执行，随机倒数一定秒数后再执行脚本，该秒数上限可以在配置文件中定义 |
| `--proxy` / `-p`      | ✔ | ✔ | 下载代理，仅适用于执行位于 GitHub 仓库的脚本，该代理固定为 [jsDelivr](https://www.jsdelivr.com/?docs=gh) 公共代理 |
| `--cookie` / `-c`     | ✔ | ✔ | 指定账号，参数后需跟`账号序号`作为参数值，多个账号用`,`隔开，账号区间用`-`连接，可以用`%`表示账号总数 |
| `--grouping` / `-g`   | ✔ |   | 账号分组，使每组账号单独运行脚本，参数后需跟`账号序号并分组`作为参数值，参数用法与 **指定账号** 相同，组与组之间用`@`隔开 |
| `--background` / `-b` | ✔ |   | 后台运行，不在前台输出脚本执行进度，不占用终端命令行 |

### 用法解读


  - #### 关于指定账号相关参数的用法示例

    :::caution 使用须知
    在某些情况下**执行脚本频率**、**单账号运行频率**过高可能会造成严重 "后果" ，例如IP被限制、账号被拉黑
    :::

    - ##### 指定第 **1** 和第 **5** 个账号执行脚本

      ```bash 
      task run example.js --cookie 1,5
      ```

    - ##### 指定前 **10** 个账号执行脚本

      ```bash
      task run example.js --cookie 1-10
      ```

    - ##### 指定第 **2** 至最后一个账号账号执行脚本

      ```bash
      task run example.js --cookie 2-%
      ```

    - ##### 指定第 **1** 和第 **5** 个账号为一组、第 **2** 和第 **6** 个账号为一组执行脚本

      ```bash
      task run example.js --grouping 1,5@2,6
      ```

      :::note 账号分组的意义？
      1.有些脚本不能有效运行全部账号，一次只能运行单个或指定数量的账号  
      2.连续执行不能满足用户助力需求  
      3.减少出现由于执行频率过高所造成的影响
      :::

  - #### 关于如何管理守护进程脚本

    - ##### 查看有哪些守护进程正在运行

      ```bash
      pm2 list
      ```
      默认有三个项目服务 `web_server` `web_terminal` `jbot`，请不要删除它们任何一个

    - ##### 停止运行

      ```bash
      pm2 delete <任务名>
      ```


## 辅助命令

### 终止执行

```bash
task stop <name/path>
```
终止某个或某些正在运行中的脚本，根据脚本名称搜索对应的进程并立即杀死，当脚本报错死循环时建议使用此功能

### 全部执行

```bash
source runall
```
通过 `交互` 选择运行模式执行指定范围的脚本，相当于批量执行脚本，时间较长不要盲目使用

### 列出脚本

```bash
task list
```
查看本地有哪些可以运行的脚本

- #### 列出指定路径下的脚本（扩展用法）

    可显示脚本修改时间和脚本文件大小

    ```bash
    task list <path>
    ```
    `<path>` 支持用 `.` 或 `./` 表示当前目录和用 `../` 表示上一层目录

## 关于执行脚本所需的依赖

这部分内容与项目命令无关，用于解决脚本运行时缺少依赖的问题

:::info 你需要知道的
当脚本报错提示 `need module xxx` 类似字样说明缺少脚本运行所需的依赖，看见 `module` 字样应立即联想到模块上  
如果缺少的依赖中带有 `/` 则表示本地依赖文件，一般开发者都会提供相关组件，注意与安装模块区分开不要弄混
:::

<Tabs>
  <TabItem value="JavaScript" label="*.js / *.ts" default>

  适用于 JavaScript 和 TypeScript 脚本

  ```bash
  npm install -g <模块名>
  ```

  你也可以到脚本所在根目录使用局部安装命令 `npm install <模块名>`，你也可以使用 `pnpm` 或 `yarn` 来安装

  </TabItem>
  <TabItem value="Python" label="*.py">

  适用于 Python 脚本

  ```bash
  pip3 install <依赖名>
  ```

  </TabItem>
</Tabs>
