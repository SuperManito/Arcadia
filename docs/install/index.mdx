本项目依托容器技术在虚拟环境中运行，因此需要安装容器服务才可以部署  
如果你不了解该项技术或者没有安装相关服务，那么请先阅读 [如何安装容器服务](/docs/install/container-service)

由于运行环境复杂程度较高，目前没有计划适配更多部署方式

***

## 启动你的容器

<Tabs>
<TabItem value="cli" label="CLI - 命令行">

```bash
docker run -dit \
--name arcadia `# 容器名` \
--hostname arcadia `# 主机名` \
--network bridge `# 容器网络类型` \
--restart always `# 开机自启` \
-p 5678:5678 `# 端口映射，"主机端口:容器端口"` \
-v /opt/arcadia/config:/root/config `# 配置文件的主机挂载目录` \
-v /opt/arcadia/log:/root/log `# 日志文件的主机挂载目录` \
-v /opt/arcadia/scripts:/root/scripts `# 个人脚本的主机挂载目录` \
-v /opt/arcadia/repo:/root/repo `# 脚本仓库的主机挂载目录` \
-v /opt/arcadia/raw:/root/raw `# 扩展脚本的主机挂载目录` \
-e ENABLE_ALL_ENV=true `# 是否在容器启动时预装环境包与常用模块` \
-e ENABLE_TG_BOT=true `# 是否在容器启动时开启电报机器人服务` \
supermanito/arcadia:latest
```

</TabItem>
<TabItem value="compose" label="Compose - 编排">

- ### 新建 YAML 文件

  ```bash
  mkdir -p /opt/arcadia && cd /opt/arcadia
  vim docker-compose.yaml
  ```

- ### 编辑内容

  ```yaml
  version: "2.0"
  services:
    arcadia:
      image: supermanito/arcadia:latest  # 镜像名，可以使用阿里云仓库的备用镜像
      container_name: arcadia  # 容器名（可随意更改）
      hostname: arcadia  # 主机名（可随意更改）
      restart: always  # 开机自启
      tty: true
      network_mode: bridge  # 容器网络类型，如果是旁路由可能需要切换为 host 类型（桥接），默认为 NAT
      ports:
        - 5678:5678  # 端口映射，格式为 "主机端口:容器端口"，主机端口号可自定义，容器端口用来访问控制面板不可修改
      volumes:
        - /opt/arcadia/config:/root/config       # 定义配置文件的主机挂载目录
        - /opt/arcadia/log:/root/log             # 定义日志文件的主机挂载目录
        - /opt/arcadia/scripts:/root/scripts     # 定义个人脚本的主机挂载目录
        - /opt/arcadia/repo:/root/repo           # 定义脚本仓库的主机挂载目录
        - /opt/arcadia/raw:/root/raw             # 定义扩展脚本的主机挂载目录
      environment:
        - ENABLE_ALL_ENV=true             # 是否在容器启动时预装环境包与常用模块
        - ENABLE_TG_BOT=true              # 是否在容器启动时开启电报机器人服务
  ```

- ### 启动容器

  ```bash
  docker-compose up -d
  ```

</TabItem>
</Tabs>

:::info 命令帮助
1. 如果想让容器内文件映射在当前目录可将默认的 `/opt/arcadia` 修改为 `$(pwd)`，可自定义映射路径  
2. 映射目录必须映射第一行的 `config` 配置文件主机挂载目录，其它目录可以不映射  
3. 如果你正在旁路由则可能需要切换为 **host** 类型（桥接），找到 `network` 字样更改即可  
4. 不可以更改参数中 `:` 右边的内容，否则会影响一些服务的正常运行  
5. 提示 `-bash: docker：Command not found/未找到命令`  ? 请先阅读 [如何安装容器服务](/docs/install/container-service)
:::

:::tip
上方的选项卡分别对应两种启动方式，不要重复执行二选一即可，新手推荐使用命令行方式部署
:::

项目容器镜像基于 [Alpine Linux](https://www.alpinelinux.org) 构建，如有其它使用需求自行查阅相关资料

## 检查启动状态

```bash
docker logs -f arcadia
```

:::info 须知
1. 请认真查看容器启动后的初始化进度并 **等待其运行完毕**，当输出 **容器启动成功** 字样时方可通过 `Ctrl + C` 退出查看  
2. 如果报错导致容器没有启动成功那么请自行检查原因，绝大多数问题都是你的网络环境所导致，自行判断不懂就百度翻译
:::

## 开始使用

访问 [http://localhost:5678](http://localhost:5678) 进入后台管理面板

:::tip
如果你使用的网络环境处于公网，请更改容器面板默认主机映射端口以及用于认证的用户名和密码，同时留意上次登录信息  
提高网络安全防范意识，尽量不要将你的面板完全暴露在公网，若遭遇不法分子入侵，作者对于由此引起的任何后果概不负责
:::

无法访问？[点击此处](/docs/panel#%E6%8E%92%E9%99%A4%E9%9D%A2%E6%9D%BF%E6%95%85%E9%9A%9C)排除故障

此外，项目支持强大的命令行指令，具体请翻阅 CLI 文档
