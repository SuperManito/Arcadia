开放接口

过时的内容

OpenApi 相关内容将在未来作出调整，会随同加载用户环境变量功能一起重做，注意持续关注我们的最新变动

接口说明​

服务端已开启 CORS 跨域支持，使用 HTTP Status Code 标识状态

  • 接口基准地址​

    http://localhost:5678/openApi

  • 需要授权的 API​

    API认证统一使用 Token 认证

    需要在 请求头（Header） 或 请求地址参数（URL） 中使用 api-token 字段并提供 openApiToken 的值

    • 获取 openApiToken 的方法​

      arcadia service info | grep ApiToken

      此 Token 初始值为 88888888 ，每次修改面板登录认证信息都会随机一个新的 32 位字符串

  • 业务代码说明​

    状态码含义说明
    0fail请求失败
    1success请求成功
    403/4403openApi 认证失败token错误

  • 通用返回内容​

    数据返回格式统一使用 JSON

    参数名参数说明
    code业务代码
    data返回结果
    msg结果消息
    desc结果描述

内置接口​

添加/更新账号数据​

POST/updateCookie​

请求​
名称类型必填描述
cookiestring是cookie 内容
userMsgstring否备注
响应​
  • ContentType application/json
名称类型描述
datanumber提交后服务器中现存的 cookie 数量
示例
{
  "code": 1,
  "data": 1,
  "msg": "success"
}

WSKEY & Cookie 二合一​

POST/addOrUpdateAccount​

请求​
名称类型必填描述
ptPinstring是pt_pin 的值
ptKeystring否pt_key 的值
wsKeystring否ws_key 的值
remarksstring否备注
信息

ptKey 和 wsKey 至少需要传入其中任意一个参数，即账号不能为空

响应​
  • ContentType application/json
名称类型描述
cookieCountnumber提交后服务器中现存的 cookie 数量
accountCountnumber提交后服务器中现存的 wskey 数量
示例
{
  "code": 1,
  "data": {
    "cookieCount": 1,
    "accountCount": 1
  },
  "msg": "success"
}

删除账号​

POST/cookie/delete​

请求​
名称类型必填描述
ptPinsarray是由pt_pin的值组成的数组,例 ["pin1","pin2"]
响应​
  • ContentType application/json
名称类型描述
cookieCountnumber删除后服务器中现存的 cookie 数量
accountCountnumber删除后服务器中现存的 wskey 数量
deleteCountnumber此次删除的 cookie 数量
示例
{
  "code": 1,
  "data": {
    "cookieCount": 1,
    "accountCount": 1,
    "deleteCount": 1
  },
  "msg": "success"
}

查看账号数量​

GETcount​

请求​

无参数

响应​
  • ContentType application/json
名称类型描述
cookieCountnumber服务器当前现存的 cookie 数量
accountCountnumber服务器当前现存的 wskey 数量
示例
{
  "code": 1,
  "data": {
    "cookieCount": 1,
    "accountCount": 1
  },
  "msg": "success"
}

查看账号信息列表​

GETlist​

请求​

无参数

响应​
  • ContentType application/json
名称类型描述
idnumber在配置文件中的序号
ptPinstringpt_key 的值
lastUpdateTimestring最近一次的更新时间
remarkstring备注
示例
{
  "code": 1,
  "data": [
    {
      "id": 1,
      "ptPin": "user_α",
      "lastUpdateTime": "2023-01-01 00:00:00",
      "remark": "xxx",
    },
    ...
  ],
  "msg": "success"
}

用户自定义接口​

目前后端使用 Node.js® 编写，所以如果你想使用这个功能则需要写一些 JavaScript 代码

编写规范

将代码存放在 config 目录下的 extra_server.js 脚本（自行创建）中，重启面板服务后生效，注意届时检查日志进行确认
脚本需要导出唯一函数名，已固定函数传参详见下方示例，接口框架使用 Express v4，更多编写方法请查阅其文档

请参考下方的简单示例

extra_server.js
async function Main(api, API_STATUS_CODE, logger) {
    // api express框架实例
    // API_STATUS_CODE 接口响应封装模板
    // logger 日志组件，支持分级（info、warn、debug、error、fatal）
    api.get('/test1', function (request, response) {
        // queryParams = request.query
        try {
            const msg = '/api/open/test1 GET 接口请求成功'
            response.send(API_STATUS_CODE.okData(msg)) // 请求成功
            logger.info(msg)
        } catch (e) {
            response.send(API_STATUS_CODE.fail(e.message)) // 请求失败
        }
    })
    api.post('/test2', function (request, response) {
        // bodyParams = request.body
        try {
            const msg = '/api/open/test2 POST 接口请求成功'
            response.send(API_STATUS_CODE.okData(msg)) // 请求成功
            logger.info(msg)
        } catch (e) {
            response.send(API_STATUS_CODE.fail(e.message)) // 请求失败
        }
    })
}

module.exports = Main

该功能使用下一代 Open Api 的接口规范，路径为 /api/open，目前还没有正式应用
