## 运行命令

<POST>/runCmd</POST>

### 请求

<APITable>

| 名称 | 类型 | 必填 | 描述 |
| :-: | :-: | :-: | --- |
| `cmd` | `string` | 是 | 命令 |

</APITable>

### 响应

```json title="示例"
"aBcDeFgHiJkLmNoPqRsTuVwXyZ"
```

### 获取实时日志

需要与 `/api/ws` 接口建立 WebSocket 连接，然后监听消息过滤 `runId` 临时id参数，然后判断 `over` 运行状态参数是否结束监听，具体详见前端业务实现

## 获取命令的运行状态

<GET>/runCmdStatus</GET>

### 请求

<APITable>

| 名称 | 类型 | 必填 | 描述 |
| :-: | :-: | :-: | --- |
| `id` | `string` | 是 | 任务临时id |

</APITable>

### 响应

```json title="示例"
true
```

:::info 注释
响应布尔值，`true` 代表任务正在运行中，`false` 代表任务已结束运行或不存在
:::
