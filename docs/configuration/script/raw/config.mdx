本篇介绍配置远程脚本的方法以及一些相关的配置项

:::tip 编辑配置文件
你可以在后台管理面板点击 `编辑配置` - `下拉选择 sync_config.yml` 来在线编辑
:::

## 主要配置项

```mdx-code-block
<APITable>
```

| 名称 | 必填 | 类型 | 默认值 | 描述 |
| :-: | :-: | :-: | :-: | --- |
| `name` | 是 | `string` | 无 | 仓库名称 |
| `url` | 是 | `string` | 无 | 仓库远程地址 |
| `cronSettings` | 否 | `object` | 无 | 定时任务设置 |
| `updateTaskList` | 否 | `boolean` | `false` | 是否为该配置启用定时任务（默认不启用） |

```mdx-code-block
</APITable>
```

### 填法示例

```yaml
raw:
  - name: ""
    url: "https://gihub.com/User1/Repo1/raw/master/template.js"
    cronSettings:
      updateTaskList: true
  - name: ""
    url: "https://gihub.com/User2/Repo2/raw/master/template.py"
```

导入前请先确认目标脚本中是否含有 `Crontab 表达式` ，如若没有或者未识别到那么将随机指定一个每天执行一次的定时，你可以在之后使用后台管理面板来修改

## 删除已配置的扩展脚本

手动删除配置文件中的相关变量后更新即可，更新后会自动删除脚本和定时无需手动删除
